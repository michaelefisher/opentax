    **Step 1: Extend ReturnContext interface** in `core/validation/types.ts`:
    Add `fieldArray(xmlName: string): readonly unknown[]` method to the ReturnContext interface.
    This returns the raw field value as an array — if the pending dict value is already an array (from mergePending promoting scalars), return it directly. If it is a single value, wrap in `[value]`. If absent/null/empty, return `[]`.

    **Step 2: Implement fieldArray in context.ts**:
    In `createReturnContext()`, add the `fieldArray` method to the returned object. Use `resolveField(xmlName)` internally. Logic:
    ```
    const v = resolveField(xmlName);
    if (v === undefined || v === null || v === "") return [];
    if (Array.isArray(v)) return v;
    return [v];
    ```

    **Step 3: Add combinators to predicates.ts** under a new section `// --- Cross-Instance Combinators ---`:

    `forEach(xml: string, predicate: (value: unknown) => boolean): RuleCheck`
    - Gets `ctx.fieldArray(xml)`, returns `items.every(predicate)`.
    - Note: The predicate here is a raw `(value: unknown) => boolean` since it operates on individual array elements, not a ReturnContext. This differs from RuleCheck.

    Actually, rethinking: the forEach combinator should work with the DSL's existing predicate pattern. The cleanest API is:

    `forEach(xml: string, itemCheck: RuleCheck): RuleCheck`
    - For each element in `ctx.fieldArray(xml)`, create a synthetic single-item ReturnContext where that field resolves to just that element, then run itemCheck against it.
    - But creating synthetic contexts is expensive. Simpler approach:

    `forEach(xml: string, itemPredicate: (value: unknown) => boolean): RuleCheck`
    - Gets `ctx.fieldArray(xml)`, returns `items.length === 0 || items.every(itemPredicate)`.
    - For validEIN usage: `forEach("EmployerEIN", (v) => validEINValue(String(v ?? "")))` where validEINValue is a helper.

    Actually simplest: make forEach work with a raw value predicate, and provide common value predicates:

    ```typescript
    /** Every element of an array field must satisfy the item predicate. Passes on empty. */
    export const forEach = (xml: string, itemPred: (v: unknown) => boolean): RuleCheck =>
      (ctx) => {
        const items = ctx.fieldArray(xml);
        return items.every(itemPred);
      };
    ```

    `everyItem(formType: string, check: RuleCheck): RuleCheck`
    - This is for multi-instance forms. Since the pending dict stores one dict per nodeType (not per instance), this combinator checks that the single merged dict satisfies the check. For rules like "if two Forms X, their SSNs must differ", use `allDistinct` on the SSN field instead.
    - Actually, since the executor merges all instances into one dict (promoting to arrays), `everyItem` is equivalent to `forEach` on the relevant field. So `everyItem` should be:

    ```typescript
    /** All values of an array-valued field must satisfy predicate. Alias for forEach. */
    export const everyItem = forEach;
    ```

    Wait, re-reading the ROADMAP more carefully, the semantics are:
    - `forEach(field, predicate)` — iterate over array values of a single field
    - `everyItem(predicate)` — check that a predicate holds for every item in a repeating group

    Given the pending dict structure (arrays from mergePending), both are essentially the same. Let me provide both with clear semantics:

    ```typescript
    export const forEach = (xml: string, itemPred: (v: unknown) => boolean): RuleCheck =>
      (ctx) => ctx.fieldArray(xml).every(itemPred);

    export const everyItem = (xml: string, itemPred: (v: unknown) => boolean): RuleCheck =>
      (ctx) => ctx.fieldArray(xml).every(itemPred);
    ```

    `sumOfAll(xml: string): number` — helper that sums array elements:
    ```typescript
    export const sumOfAll = (target: string, source: string): RuleCheck =>
      (ctx) => {
        const items = ctx.fieldArray(source);
        const sum = items.reduce((s, v) => s + (typeof v === "number" ? v : Number(v) || 0), 0);
        return Math.abs(ctx.num(target) - sum) < 0.01;
      };
    ```

    `allDistinct(xml: string): RuleCheck` — all array values are unique (for SSN non-duplicate rules):
    ```typescript
    export const allDistinct = (xml: string): RuleCheck =>
      (ctx) => {
        const items = ctx.fieldArray(xml).map((v) => String(v ?? "").replace(/\D/g, "")).filter(Boolean);
        return new Set(items).size === items.length;
      };
    ```

    Also export a helper for use with forEach:
    ```typescript
    export const isValidEINValue = (v: unknown): boolean => {
      const s = String(v ?? "").replace(/\D/g, "");
      if (!s) return true;
      if (s.length !== 9) return false;
      return VALID_EIN_PREFIXES.has(s.substring(0, 2));
    };
    ```
