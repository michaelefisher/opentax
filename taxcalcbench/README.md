# taxcalcbench

Accuracy benchmark for the `tax` engine — 97 TY2025 scenarios with independently
computed correct values. Passes when every engine output is within $5 of the correct
value for total tax, refund, and amount owed.

## What's in here

```
taxcalcbench/
  run_benchmark.ts    # Runs all cases through the engine, compares to correct.json
  run_case.ts         # Runs a single case through the engine, saves output.json
  run_all.ts          # Regenerates output.json for every case (bulk engine run)
  harness/
    state.json        # Active task state: pass/fail counts, root causes, phase
    progress.md       # Append-only log of harness runs and outcomes
  cases/
    01-single-w2-minimal/
      input.json      # Tax return inputs (forms + filing info)
      correct.json    # Ground truth computed by tax2025.ts
      output.json     # Last engine output (generated by run_case.ts / run_benchmark.ts)
    02-single-w2-basic/
    ...
    51-single-senior-1099r-f1040es/
```

## Cases

97 scenarios covering the common return types:

| Range | Filing status | Key features tested |
|-------|--------------|---------------------|
| 01–10 | Single | W-2 only, high income, unemployment, interest, capital gains, Schedule C, student loan, senior, two employers |
| 11–17 | MFJ | Both W-2, children/CTC, unemployment, dividends, Schedule C + QBI, retirement + SSA, excess SS tax |
| 18–20 | HOH | Child/CTC, EITC, dependent care credit |
| 21–31 | Mixed | Additional Medicare Tax, ACTC, LTCG 0% bracket, blind filer, senior + SE income |
| 32–51 | Extended | Schedule C loss, 1099-R, AOTC, marketplace/1095-A, educator expense, estimated tax, EITC no children, 401(k), tips, multiple 1099-R, QBI, senior + f1040es |

**Pass criteria:** engine value within $5 of the correct value for:
- `line24_total_tax`
- `line35a_refund`
- `line37_amount_owed`

AGI and taxable income are shown in the output for debugging but do not affect pass/fail.

## How to run

### Prerequisites

```bash
# Deno required (no Python or shell dependencies)
# Build the tax binary (from repo root)
deno compile --output tax cli/main.ts
```

### Run the full benchmark

```bash
cd taxcalcbench
deno run --allow-read --allow-write --allow-run run_benchmark.ts
```

Sample output:

```
Case                              AGI      Taxable     TotalTax     Payments       Refund         Owed
------------------------------ ------------ ------------ ------------ ------------ ------------ ------------
01-single-w2-minimal               15000            0            0          314          314            0  PASS
02-single-w2-basic                 50000        35000         3962         5001         1039            0  PASS
...
────────────────────────────────────────────────────────────────────────────────
Results: 51 PASS  0 FAIL  out of 51 cases
Green = within $5 of correct value.  Red = engine differs from expected.
```

### Run a single case

```bash
deno run --allow-read --allow-write --allow-run run_case.ts cases/02-single-w2-basic/
```

Saves engine output to `cases/02-single-w2-basic/output.json` and prints the
summary.

### Regenerate all output.json files

```bash
deno run --allow-read --allow-write --allow-run run_all.ts
```

## How cases are structured

Each `input.json` is a list of forms in the same format the engine CLI accepts:

```json
{
  "year": 2025,
  "scenario": "Human-readable description",
  "forms": [
    { "node_type": "start",  "data": { "general": { "filing_status": "single", ... } } },
    { "node_type": "w2",     "data": { "box1_wages": 50000, "box2_fed_withheld": 6000, ... } },
    { "node_type": "f1099g", "data": { "box_1_unemployment": 4200, ... } }
  ]
}
```

Each `correct.json` contains the ground-truth values computed by `tax2025.ts`,
keyed by 1040 line numbers:

```json
{
  "correct": {
    "line11_agi": 50000,
    "line15_taxable_income": 35000,
    "line24_total_tax": 3962,
    "line33_total_payments": 5001,
    "line35a_refund": 1039,
    "line37_amount_owed": 0
  }
}
```

## Adding a new case

Use `/tax-cases` to generate IRS-sourced cases automatically, or create manually:

1. Create `cases/NN-description/input.json` following the format above.
2. Write `correct.json` with values sourced directly from an IRS publication (VITA Pub 4491, Pub 17, MeF test cases). Do not compute values — cite the IRS source.
3. Run `deno run --allow-read --allow-write --allow-run run_benchmark.ts` to verify the engine result.

## 2025 tax parameters

| Parameter | Value |
|-----------|-------|
| Standard deduction — Single / HOH | $15,000 / $22,500 |
| Standard deduction — MFJ | $30,000 |
| Senior/blind add-on — Single/HOH | +$2,000 per factor |
| Senior/blind add-on — MFJ | +$1,600 per factor |
| SS wage base | $176,100 |
| EITC max (0 / 1 / 2 / 3+ children) | $649 / $4,328 / $7,152 / $8,046 |
| EITC MFJ phase-out start (1–3 children) | $30,323 |
| CTC per child | $2,000 |
| ACTC rate | 15% of earned income over $2,500, up to $1,700/child |
| LTCG 0% threshold — Single / MFJ | $48,350 / $96,700 |
| QBI deduction | 20% of lesser of (QBI, taxable income before QBI) |
| Additional Medicare Tax | 0.9% on wages/SE over $200k (Single) / $250k (MFJ) |

## EITC earned income definition

Per IRC §32(c)(2)(A)(ii), earned income for EITC includes both wages and net
earnings from self-employment. The reference calculator (`tax2025.ts`) and the
engine both use `wages + max(0, schedule_c_net_profit)` as the EITC earned
income base. The Schedule C node wires its aggregate net profit to the EITC
node via the `se_net_profit` field so the two stay in sync.

Cases with Schedule C income that exceeds the EITC phase-out threshold (e.g.,
case 23) will correctly receive $0 EITC even if wages alone would have
qualified; the correct.json files reflect this.
