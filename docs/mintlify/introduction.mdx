Tax calculation is a solved problem — every number on a 1040 follows published IRS formulas. But the software that does it has always been closed, expensive, and centralized.

**tax-engine** is the open-source alternative: a single binary that runs the full Form 1040 calculation on your machine, follows the same IRS math, and speaks the same language as the AI tools you already use.

## What it is

A **headless CLI engine** for federal income tax. It takes structured input (wages, dividends, deductions) and produces structured output (every computed line of the 1040). No UI, no cloud, no accounts — just a binary that does IRS math.

The CLI is the interface. AI is the UX layer on top.

## What it covers

Form 1040 (Tax Year 2025), including:

- All income types — wages, interest, dividends, self-employment, retirement, rental, farm, gambling
- All standard schedules — A, C, E, F, H, J, R, K-1 variants
- Credits — child tax credit, dependent care, education, energy, EV, business
- Deductions — mortgage interest, student loans, IRA, SEP, educator expenses, QBI
- Capital transactions — Form 8949, installment sales, like-kind exchanges
- Foreign and special situations — FBAR, PFIC, injured spouse, community property

131 input nodes. 184 total nodes in the dependency graph.

## How it fits into AI workflows

The binary accepts JSON on the command line and returns JSON. That's it. Any AI that can run a subprocess can use it:

```
AI receives prompt → calls tax binary → gets results → explains them to user
```

No MCP, no plugins, no API keys — just a CLI call that any AI can make. See [Use with AI](/use-with-ai/overview) for examples with Claude, ChatGPT, and Gemini.

## Design principles

**Local first.** Your tax data never leaves your machine. The binary has no network access beyond optional IRS form downloads.

**Open source.** MIT licensed. The calculation logic is auditable, forkable, and improvable by the community.

**Schema first.** Every input is validated against a Zod schema before computation. Bad data fails loudly before numbers are touched.

**Pure functions.** Each node is a pure transformation: validated input in, typed outputs out. No state, no side effects, no surprises.

**IRS-accurate.** 97 real-world scenarios sourced directly from IRS publications, validated against IRS-computed values. Business rules run the full MeF validation suite.
